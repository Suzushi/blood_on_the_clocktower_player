# 🩸 血染钟楼 - 在线网游

这是一个基于 Web 的《血染钟楼》（Blood on the Clocktower）桌面游戏。它旨在提供一个线上游戏环境，让玩家能够在不依赖线下游戏环境的情况下，与其他玩家互动。

- **🤖 自动化夜间流程**：
  - 自动计算并显示正确的夜间唤醒顺序。
  - **智能信息生成**：自动计算洗衣妇、共情者、厨师、调查员等角色的查验信息，甚至考虑了**中毒/醉酒**的影响。

  - 内置提名与投票计算器，自动判断处决结果。
  - 自动处理复活、死亡、更改状态。
- **🧩 代码结构升级**：
  - 平衡参数已独立到 `config/balance.py`。
  - 路由统一收敛到玩家端接口链路。
  - 信息生成逻辑开始下沉至 `services/`，便于后续多剧本扩展。

## 🚀 快速开始

### 环境要求

- Python 3.8 或更高版本

### 安装步骤

1. **克隆或下载本项目**到本地。
2. **准备虚拟环境**：
   本项目当前默认使用 `~/.venv` 作为共享 Python 环境。
   ```bash
   uv venv ~/.venv
   uv pip install -r requirements.txt --python ~/.venv/bin/python
   ```

### 启动游戏

1. **运行程序**：
   ```bash
   ~/.venv/bin/python main.py
   ```
2. **访问界面**：
   打开浏览器，访问地址：`http://127.0.0.1:5000/player`

   您将看到一个玩家端界面，用于与其他玩家互动。
   玩家可以自由创建游戏房间，也可以加入已有的游戏房间。
3. **加入游戏**：
   - 创建房间会自动生成房间号
   - 输入房间号，点击“加入游戏”按钮，系统会弹出界面让你选择座位。
   - 系统会自动将您加入当前游戏房间。
4. **等待其他玩家**：
   - 其他玩家也需要加入游戏，系统会自动将他们加入当前游戏房间。
   - 当所有玩家都加入后，游戏开始。

## ⚠️ 注意事项

- **数据存储**：当前版本为**内存运行模式**。**请勿关闭运行** **`main.py`** **的终端窗口**，否则当前游戏进度会丢失。
- 房间生成以后，游玩身份与座位绑定，退出房间会导致该座位无法使用。
- 夜间消息推送收游览器影响，如果没有弹窗，可以直接翻阅个人信息历史记录。
- 当前默认信任模型是“熟人局/轻部署”：
  - 不提供注册登录系统。
  - 玩家进入座位后由服务端签发 `reconnect_token`。
  - 所有玩家私有/状态类接口都必须携带该 token。
  - 房间码和 `player_id` 只用于定位房间与座位，不再单独构成授权。
- 房主控制链路继续使用 `owner_token`，用于开始游戏、推进阶段、强制结算等房主管理操作。

## 🔐 鉴权约定

- `无需玩家鉴权`：`find_game`、`join_game`、`reconnect`
- `必须携带 reconnect_token`：所有读取玩家私有信息、提交提名/投票、提交夜间行动、读取待处理行动、读取消息、心跳续连等 `/api/player/*` 状态接口
- 本项目当前不引入账号体系；“点开即玩”依赖的是座位绑定 + 浏览器本地保存的 `reconnect_token`
- 若未来部署到公网，仍建议配合 HTTPS 使用，否则 token 可能被链路窃取

## 🧭 接口分层

- `/api/player/*`：当前主链路。后续新增玩家功能默认从这里进入。
- `/api/game/*`：说书人管理接口 + 部分兼容/调试接口。
- 旧兼容接口如 `generate_info`、`slayer_ability` 当前保留是为了测试和历史调用稳定，不再作为新功能主入口继续扩展。
